# Основные способы создания thread safe counter

### 1. Использование synchronized методов

Это классический и простой подход. Вы создаёте класс с приватным полем int или long для хранения значения, и все методы, которые обращаются к этому полю (increment, decrement, get), помечаете ключевым словом synchronized. Когда один поток входит в такой метод, остальные потоки блокируются до его завершения. 
- Плюсы: просто и понятно. 
- Минусы: может быть узким местом при высокой конкуренции потоков.

### 2. Использование AtomicInteger/AtomicLong из пакета java.util.concurrent.atomic
   
Это более современный и эффективный подход. Вместо обычного int вы используете AtomicInteger, который имеет встроенные потокобезопасные методы вроде incrementAndGet(), decrementAndGet(), get(). Под капотом это использует внутренние механизмы синхронизации (CAS - Compare-and-Swap операции), которые работают быстрее, чем synchronized. 
- Плюсы: лучшая производительность, меньше блокировок. 
- Минусы: немного менее интуитивно на первый взгляд.

### 3. Использование ReentrantReadWriteLock или обычного Lock (ReentrantLock)

Это более гибкий подход из пакета java.util.concurrent.locks. Вы создаёте объект lock и оборачиваете критические секции кода в try-finally блоки с явным захватом и освобождением lock'а. Преимущество в том, что с ReadWriteLock вы можете позволить нескольким потокам одновременно читать значение, но только одному писать. 
- Плюсы: большая гибкость, возможность различать операции чтения и записи. 
- Минусы: более многословно, нужно быть аккуратнее с управлением lock'ом.
