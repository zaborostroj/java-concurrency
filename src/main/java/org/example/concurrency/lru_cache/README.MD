# Реализация LRU Cache

`LruCache` — это реализация кэша с вытеснением давно неиспользуемых элементов (Least Recently Used). 

## Принцип работы

Данная реализация использует два основных компонента:
1.  **ConcurrentHashMap**: Обеспечивает быстрый доступ к элементам по ключу ($O(1)$).
2.  **Двусвязный список**: Хранит порядок использования элементов. В начало списка (`head`) перемещаются недавно использованные или только что добавленные элементы, а из конца списка (`tail`) вытесняются наиболее старые при достижении лимита емкости.

## Особенности реализации

-   **Потокобезопасность**: Для синхронизации доступа к двусвязному списку и обновления размера используется `ReentrantLock` (объект `mutex`).
-   **Атомарность**: Операции `get`, `put` и `remove` защищены блокировкой, что гарантирует консистентность данных между `Map` и списком.
-   **Прерывание**: Используется `lockInterruptibly()`, что позволяет корректно обрабатывать прерывания потоков.
-   **Lombok**: Для уменьшения шаблонного кода в узле списка (`Node`) используются аннотации `@Getter`, `@Setter`, `@AllArgsConstructor` и `@EqualsAndHashCode`.

## Основные методы

-   `V get(K key)`: Возвращает значение по ключу и перемещает узел в начало списка.
-   `void put(K key, V value)`: Добавляет или обновляет значение. Если ключ новый и достигнута емкость (`capacity`), удаляет наименее используемый элемент (из хвоста списка).
-   `void remove(K key)`: Удаляет элемент по ключу.
-   `int getSize()`: Возвращает текущее количество элементов в кэше.
